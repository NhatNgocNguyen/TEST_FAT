✅ STEP 1 — Create Feature Group in Vertex AI

Your BigQuery table exists, but Feature Monitoring works on Feature Groups, not directly on BigQuery tables.

1️⃣ Go to
Google Cloud Console
→ Vertex AI
→ Feature Store

Make sure you're using Feature Store (v2).

2️⃣ Click Create Feature Group

Fill in:

Basic info

Name: customer-churn-fg

Region: same as BigQuery dataset (e.g. us-central1)

Click Continue

3️⃣ Select Source

Choose:

✔ BigQuery table

Select:

ext-bidv-eda-internal.test.customer_churn_demo

Click Continue

4️⃣ Configure Entity

Entity ID column:

customer_id

Timestamp column:

feature_timestamp

Click Continue

5️⃣ Select Features

Select ALL except:

❌ customer_id
❌ feature_timestamp

So select:

age

gender

tenure_months

avg_monthly_spend

num_support_tickets

contract_type

payment_method

churned

Click Create

Wait until Feature Group is ready.

✅ STEP 2 — Create Feature Monitor

Now we configure monitoring.

1️⃣ Inside Feature Group page

Go to:

Monitoring tab

Click:

✔ Create Monitor

2️⃣ Configure Monitor
Monitor Name
customer-churn-monitor
Select Features to Monitor

Choose:

age

tenure_months

avg_monthly_spend

num_support_tickets

contract_type

payment_method

(You can include churned too)

Baseline Dataset

Choose:

✔ Use existing feature data (current table snapshot)

This becomes baseline distribution.

Drift Detection Threshold

For numeric features:

Keep default OR set 0.05

For categorical:

Keep default

Monitoring Schedule

For demo:

✔ Run on demand

Click Create

✅ STEP 3 — Create Drift Data (Simulate Distribution Shift)

Now we create new data that is very different.

Go to:

BigQuery → Query Editor

Run this:

INSERT INTO `ext-bidv-eda-internal.test.customer_churn_demo`
VALUES
('C009', 70, 'Male', 72, 150.0, 0, 'Two year', 'Credit card', 0, CURRENT_TIMESTAMP()),
('C010', 68, 'Female', 65, 170.0, 0, 'Two year', 'Credit card', 0, CURRENT_TIMESTAMP()),
('C011', 75, 'Male', 80, 200.0, 1, 'Two year', 'Bank transfer', 0, CURRENT_TIMESTAMP()),
('C012', 72, 'Female', 90, 180.0, 0, 'Two year', 'Credit card', 0, CURRENT_TIMESTAMP());

Notice:

age now ~70 instead of ~30–40

spend much higher

mostly Two year contracts

This creates strong drift.
